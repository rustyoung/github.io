zookeeper基本命令

使用zkCli连接：
zkCli -server 127.0.0.1:2181
help命令

列出所有命令

[zk: localhost:2181(CONNECTED) 0] help
ZooKeeper -server host:port cmd args
    stat path [watch]
    set path data [version]
    ls path [watch]
    delquota [-n|-b] path
    ls2 path [watch]
    setAcl path acl
    setquota -n|-b val path
    history 
    redo cmdno
    printwatches on|off
    delete path [version]
    sync path
    listquota path
    rmr path
    get path [watch]
    create [-s] [-e] path data acl
    addauth scheme auth
    quit 
    getAcl path
    close 
    connect host:port
ls命令

获取路径下的节点信息，注意此路径为绝对路径

[zk: localhost:2181(CONNECTED) 1] ls /
[zookeeper]
create命令

创建节点，其中-s为顺序充点，-e临时节点(临时节点连接断开时数据就丢掉)

[zk: localhost:2181(CONNECTED) 3] create /zk test
Created /zk
[zk: localhost:2181(CONNECTED) 4] ls /
[zk, zookeeper]

//顺序充点
[zk: localhost:2181(CONNECTED) 10] create -s /a 1
Created /a0000000008
[zk: localhost:2181(CONNECTED) 11] create -s /b 2
Created /b0000000009
get命令

获取节点信息，注意节点的路径皆为绝对路径

[zk: localhost:2181(CONNECTED) 5] get /zk
test //节点数据信息
cZxid = 0xb //节点创建时的zxid
ctime = Sat Feb 10 16:05:21 EST 2018 //节点创建时间
mZxid = 0xb //节点最近一次更新时的zxid
mtime = Sat Feb 10 16:05:21 EST 2018 //节点最近一次更新的时间
pZxid = 0xb //该节点的子节点（或该节点）的最近一次 创建 / 删除 的时间
cversion = 0 //子节点数据更新次数
dataVersion = 0 //本节点数据更新次数
aclVersion = 0 //点ACL(授权信息)的更新次数
ephemeralOwner = 0x0 //如果该节点为临时节点,ephemeralOwner值表示与该节点绑定的session id. 如果该节点不是临时节点,ephemeralOwner值为0
dataLength = 4 //节点数据长度
numChildren = 0 //子节点个数
set命令

设置节点的数据

[zk: localhost:2181(CONNECTED) 6] set /zk hehe
[zk: localhost:2181(CONNECTED) 7] get /zk
hehe
delete&rmr命令

删除节点命令

[zk: localhost:2181(CONNECTED) 8] delete /zk
[zk: localhost:2181(CONNECTED) 9] get /zk
Node does not exist: /zk

//注意：delete不可删除有子节点的节点，但是rmr命令可以删除
[zk: localhost:2181(CONNECTED) 42] ls /zk
[aa]
[zk: localhost:2181(CONNECTED) 43] delete /zk
Node not empty: /zk
[zk: localhost:2181(CONNECTED) 44] rmr /zk
[zk: localhost:2181(CONNECTED) 45] get /zk
Node does not exist: /zk
stat命令

查看节点状态信息

close命令

断开客户端与服务端的连接

ls2命令

ls2为ls命令的扩展，比ls命令多输出本节点信息

history命令

列出最近的历史命令

setAcl命令

设置节点Acl。

Acl由大部分组成：1为scheme，2为user，3为permission，一般情况下表示为scheme:id:permissions。
getAcl命令

获取节点Acl

addauth命令

节点认证

sync命令

强制同步

由于请求在半数以上的zk server上生效就表示此请求生效，那么就会有一些zk server上的数据是旧的。sync命令就是强制同步所有的更新操作
redo命令

再次执行某命令

如redo 10，其中10为命令ID，需与history配合使用
setquota命令

设置子节点个数和数据长度配额

如setquota –n 4 /zookeeper/node 设置/zookeeper/node子节点个数最大为4
setquota –b 100 /zookeeper/node 设置/zookeeper/node节点长度最大为100
listquota命令

显示配额

delquota命令

删除配额，-n为子节点个数，-b为节点数据长度
